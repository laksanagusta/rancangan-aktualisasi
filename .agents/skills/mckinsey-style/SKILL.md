# McKinsey Presentation Style

Apply the McKinsey & Company consulting presentation structure and design philosophy to create clear, persuasive, and executive-ready slide decks. This skill transforms content into structured consulting-grade presentations.

## When to Use This Skill

- User requests a "McKinsey-style" or "consulting-style" presentation
- User needs to present strategy recommendations to senior executives
- User wants a data-driven, minimalist presentation structure
- User mentions Pyramid Principle, MECE, or action titles
- User needs an executive summary deck or strategy deck

## Core Principles

These six principles are **non-negotiable** in every McKinsey-style presentation:

### 1. The Pyramid Principle (Top-Down Communication)
Present the **main conclusion first**, then supporting arguments, then evidence.

```
Level 1: Main Conclusion / Recommendation
├── Level 2: Supporting Argument A
│   ├── Level 3: Data / Evidence A1
│   └── Level 3: Data / Evidence A2
├── Level 2: Supporting Argument B
│   ├── Level 3: Data / Evidence B1
│   └── Level 3: Data / Evidence B2
└── Level 2: Supporting Argument C
    ├── Level 3: Data / Evidence C1
    └── Level 3: Data / Evidence C2
```

**Example:**
- **Conclusion:** "The company should invest in renewable energy technology"
- **Supporting Arguments:**
  1. Market Growth — renewable energy sector expanding rapidly
  2. Regulatory Incentives — government subsidies and tax breaks
  3. Corporate Responsibility — sustainability enhances brand image
- **Evidence:** Growth rate statistics, specific policy details, consumer studies

### 2. The MECE Framework
All information must be **Mutually Exclusive** and **Collectively Exhaustive**.

- **Mutually Exclusive:** Each point is distinct — no overlapping content
- **Collectively Exhaustive:** All points together cover the entire scope of the argument — no gaps

**How to apply:**
- Categorize content to avoid duplication (group similar ideas with supporting data)
- Use Gap Analysis to check for missing critical elements
- Apply hierarchy — organize from most to least impactful

### 3. Data-Driven Insights
Every recommendation and conclusion must be backed by solid data.

- **Quantitative:** Statistics, charts, graphs, numerical trends
- **Qualitative:** Expert opinions, case studies, testimonials
- Only include data that **directly supports** your point — no filler
- Present data in clear, focused formats (full-slide charts preferred)

### 4. Minimalist & Consistent Design
Clean, uncluttered slides with a consistent visual theme.

- Limited color palette aligned with brand identity
- Same fonts, colors, and layouts throughout
- No script typefaces — use clean, readable fonts
- Strong contrast between text and background
- Ample white space to reduce visual fatigue
- Clear visual hierarchy: headings > subheadings > bullet points

### 5. Logical Flow & Coherent Narrative
Structure the presentation to guide the audience seamlessly through the argument.

- Use the **SCR Framework** (Situation → Complication → Resolution)
- Each slide is a stepping stone toward the recommendation
- No tangents or unnecessary detours

### 6. Audience-Centric Approach
Tailor content and delivery to the audience's needs and expectations.

- Senior executives → brevity, conclusions first, actionable items
- Technical audience → more detailed evidence and methodology
- Mixed audience → layered depth (summary first, details in appendix)

## Slide Anatomy: The Three-Zone Layout

Every McKinsey body slide follows a strict three-zone layout:

```
┌─────────────────────────────────────────────────┐
│  ACTION TITLE                                   │
│  A complete sentence stating the slide's key    │
│  takeaway or insight                            │
├─────────────────────────────────────────────────┤
│                                                 │
│                                                 │
│                  SLIDE BODY                     │
│                                                 │
│  Charts, graphs, infographics, text, icons,     │
│  or any combination that supports the           │
│  action title                                   │
│                                                 │
│                                                 │
├─────────────────────────────────────────────────┤
│  SOURCE AND FOOTER                              │
│  Data source citation  |  Company Name  |  Page │
└─────────────────────────────────────────────────┘
```

### Action Title Rules
- **Must be a complete sentence** — not a label or topic
- States the **key takeaway** the audience should remember
- Should be readable and understandable **on its own**
- Typically bold, larger font, visually separated from body

| ❌ Bad Title | ✅ Good Action Title |
|---|---|
| "Revenue Overview" | "Revenue grew 23% YoY driven by enterprise expansion" |
| "Market Analysis" | "Southeast Asian market represents $4.2B untapped opportunity" |
| "Customer Satisfaction" | "Customer satisfaction scores declined 15pp since Q2, requiring immediate intervention" |
| "Cloud Migration" | "Developers report improved agility and resilience in cloud environments" |

### Source & Footer Rules
- Always cite data sources
- Include company/organization name
- Include page/slide number
- Footnotes for methodology or caveats go here

## Presentation Structure (Slide Order)

### 1. Title Slide
- Presentation title (concise, states the main focus)
- Client/organization name
- Date
- Consulting team names (optional)
- **Minimal design** — clean lines, consistent fonts, restrained color

### 2. Executive Summary
- High-level overview of the **entire** presentation
- Answers the most important questions **upfront**
- Summarizes: key findings, recommendations, and next steps
- Decision-makers should grasp the core message in **2-3 minutes**
- References core evidence but keeps it brief
- Detailed data appears in body slides

**Formatting guidelines for executive summary:**
- Use columns with wide gutters for multi-topic summaries
- Line spacing between paragraphs for readability
- Bold key phrases for scannability
- Despite being text-heavy, maintain clean formatting

### 3. Body Slides (Analysis & Evidence)
Choose the appropriate layout per slide based on content type:

#### Layout A: Full-Slide Graph
- One chart/graph occupies the entire slide body
- Minimal or no additional text beyond the action title
- Maximum visual impact and focus on data trends
- Best for: time series, market trends, performance metrics

#### Layout B: Infographic + Text Combination
- Infographics visualize complex ideas in digestible format
- Accompanying text provides context or actionable insights
- Side banner (≈1/3 width) with contrasting color for emphasis
- Best for: process flows, concept explanations, comparisons

#### Layout C: Full-Text Slide
- Used sparingly — only for key findings, recommendations, or next steps
- **Rules:**
  - Brevity: short sentences and bullet points
  - Hierarchy: bold headings differentiate sections
  - White space: avoid overcrowding
- Keep to ONE text-heavy slide per topic; never mix large text with charts

#### Layout D: Text + Icons
- Icons represent concepts visually, breaking up text
- Arranged in table/grid format for clear organization
- Best for: myth vs. reality, feature comparisons, process steps
- **Why it works:** simplification, visual interest, clear organization

#### Layout E: Data Dashboard
- Multiple small charts/KPIs on a single slide
- Use when summarizing large datasets
- Each element has its own mini-title
- Best for: KRI reporting, performance dashboards, multi-metric views

### 4. Conclusion / Recommendations Slide
- Summarize main points and insights (3-5 concise points)
- Provide **specific, prioritized actions**
- Include a detailed outline of immediate next steps
- Attendees must know **how to proceed** after the presentation
- Highlights signal only the most critical insights linked to recommendations

## Storyline Framework: SCR

Structure the narrative using **Situation-Complication-Resolution**:

1. **Situation (Introduction):** Frame the core issue. Set the stage. Give the audience a reason to care.
2. **Complication:** Identify challenges, urgency factors, and complications that heighten the need for action.
3. **Resolution:** Lead to the solution or recommendation. Every slide building to this point reinforces credibility.

### Storyline Quality Checklist
- [ ] Does every slide serve the central narrative?
- [ ] Can the storyline be summarized in one sentence?
- [ ] Does each slide lead logically to the next?
- [ ] Are there any tangential slides that don't support the resolution?
- [ ] Does the presentation build toward a clear, actionable conclusion?

## Design Specifications

### Typography
- **Action titles:** Bold, larger size (e.g., 20-24px equivalent)
- **Body text:** Clean sans-serif, readable at distance (e.g., 14-16px)
- **Footnotes/sources:** Smaller, muted color (e.g., 10-12px)
- **NO script typefaces** — ever
- Recommended fonts: IBM Plex Sans, Source Sans Pro, Helvetica Neue, or similar clean sans-serif

### Color Palette
- Use a **limited, cohesive palette** (3-5 colors max)
- One dominant color + 1-2 accent colors + neutrals
- McKinsey's classic palette: dark navy/teal backgrounds, white text, blue accents
- For custom brands: align with client's brand identity
- Maintain **strong contrast** for readability

### Layout & Spacing
- Generous margins and padding
- Clear visual hierarchy guiding the reader's eye
- White space as a deliberate design element
- Consistent alignment across all slides
- Grid-based layouts for structured content

## Content Writing Rules

### The "One Breath" Rule
Each bullet point or sentence should be short enough to read in a single breath.

### Text Guidelines
- Prefer bullet points over paragraphs
- Break long narratives into scannable chunks
- Bold key terms and data points
- One idea per bullet point
- Avoid jargon unless the audience expects it

### Data Presentation
- Choose the right chart type for the data story
- Label axes, units, and time periods clearly
- Highlight the key insight with callouts or annotations
- Use consistent color coding across related charts
- Source every data point

## Implementation Checklist

Before finalizing any McKinsey-style presentation, verify:

- [ ] **Pyramid Principle:** Main conclusion is stated upfront (title slide or executive summary)
- [ ] **MECE:** Arguments are mutually exclusive and collectively exhaustive
- [ ] **Action Titles:** Every body slide has a sentence-form action title
- [ ] **Three-Zone Layout:** Every body slide has action title / body / source footer
- [ ] **Data Sources:** Every data point is cited in the footer
- [ ] **SCR Storyline:** Presentation follows Situation → Complication → Resolution
- [ ] **Minimalist Design:** No clutter, consistent fonts/colors, ample white space
- [ ] **Executive Summary:** Captures the entire deck's message in 2-3 minutes
- [ ] **Conclusion Slide:** Has 3-5 key points + specific next steps
- [ ] **One Breath Rule:** All text is concise and scannable
- [ ] **Consistent Formatting:** Same visual treatment across all slides
